# Recluster — пересборка категорий кластеризацией

Полностью пересчитывает `document_type` для всех документов, но **не читает
исходные файлы вообще** — только текстовые `summary`, уже лежащие в БД. Тип
файла (`.docx`/pdf/скан) значения не имеет никогда. Подробный алгоритм —
[recluster.md](../recluster.md) (обычная документация); здесь только сжатая
диаграмма решений.

Реализация: [`recluster.py`](../../backend/app/services/recluster.py).

```
run_recluster(min_clusters, max_clusters, provider_id?)
        │
   1. отобрать документы: analysis_status=="done" И summary не пусто
        │
   2. каждый summary → _strip_for_clustering()
        (убрать даты/имена/орг/теги — оставить только "о чём документ")
        осталось < 20 симв. после чистки? ──да──► взять исходный summary целиком
        │
   3. embed (sentence-transformers, локально, бесплатно)
        │
   4. подобрать k: диапазон [k_min, k_max] из размера коллекции →
        перебрать ~8 кандидатов k → выбрать по silhouette score
        │
   5. KMeans(k)
        │
   6. для каждого кластера: 3–5 ближайших к центроиду документов-примеров
        │
   7. на КАЖДЫЙ кластер — один платный LLM-вызов (_name_cluster):
        │
        LLM вернул иконку, которая занята другим кластером ЭТОГО прогона,
        либо не входит в разрешённый список? ──да──► добавить в EXCLUDED,
                                                       повторить (до 3 раз)
        │
        все 3 попытки не удались, или свободных иконок не осталось?
        ──да──► fallback: slug="unclassified", icon="FileText"
                (кластер помечен, но ОБЩИЙ прогон не падает)
        │
        ВСЕ кластеры этого прогона свалились в fallback?
        ──да──► весь run прерывается ДО применения изменений
                (защита от "провайдер сломан целиком")
        │
   8. apply: для каждого документа —
        старый тип был содержательным (не unclassified/other) И реально
        изменился? ──да──► сохранить старый тип как тег
        новый document_type проставлен, classification_source="auto",
        manually_classified СБРАСЫВАЕТСЯ в False (recluster всегда
        перебивает ручную классификацию)
        │
   9. custom_type_icons / custom_type_names (en/fr/ru) — upsert в AppSettings,
        built-in слаги и "unclassified" не трогаются
```

## Отличие от reclassify-задач

Recluster не знает про отдельные документы во время LLM-вызовов — экономия на
масштабе (1 вызов на группу, а не на документ), но ценой того, что таксономия
типов документов **создаётся заново с нуля**, а не подгоняется под
существующую. Обычно запускается редко (реорганизация всей библиотеки), в
отличие от `reclassify_unclassified`/`reclassify_all`, которые точечно
дозаполняют пробелы.
